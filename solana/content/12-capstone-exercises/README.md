## How to Approach

1. **Read the requirements** completely before writing any code
2. **Plan your accounts and PDAs** on paper first
3. **Build incrementally** -- get one instruction working, then the next
4. **Test as you go** -- don't wait until the end
5. **Check the hints** only if you're stuck for 20+ minutes
6. **Compare with the solution outline** after you've built your version
