# Metadata and Attributes

NFT metadata lives in two places: on-chain (in the asset account) and off-chain (in a JSON file).

## On-Chain vs Off-Chain Metadata

| Data | Where | Why |
|------|-------|-----|
| Name | On-chain | Quick display, searchable |
| URI | On-chain | Points to full metadata |
| Owner | On-chain | Proves ownership |
| Collection | On-chain | Groups related assets |
| Description | Off-chain (JSON) | Too long for on-chain |
| Image | Off-chain (Arweave) | Files can't live on-chain |
| Attributes | Off-chain (JSON) or on-chain (plugin) | Flexible |

## The Metadata JSON Standard

NFT metadata follows a standard format that wallets and marketplaces understand:

```json
{
    "name": "Cosmic Dragon #42",
    "symbol": "DRAGON",
    "description": "A rare cosmic dragon generated by AI. Style: cyberpunk, Model: StableDiffusion XL.",
    "image": "https://arweave.net/IMAGE_HASH",
    "animation_url": "https://arweave.net/VIDEO_HASH",
    "external_url": "https://yourplatform.com/asset/42",
    "attributes": [
        { "trait_type": "Style", "value": "Cyberpunk" },
        { "trait_type": "Element", "value": "Fire" },
        { "trait_type": "Rarity", "value": "Legendary" },
        { "trait_type": "Generation", "value": "3" },
        { "trait_type": "Model", "value": "SDXL 1.0" }
    ],
    "properties": {
        "category": "image",
        "files": [
            {
                "uri": "https://arweave.net/IMAGE_HASH",
                "type": "image/png"
            }
        ],
        "creators": [
            {
                "address": "CREATOR_WALLET_ADDRESS",
                "share": 100
            }
        ]
    }
}
```

### Field Descriptions

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Display name |
| `symbol` | No | Short symbol |
| `description` | No | Full description |
| `image` | Yes | URL to the preview image |
| `animation_url` | No | URL to video/audio/3D content |
| `external_url` | No | Link to your platform page |
| `attributes` | No | Array of trait/value pairs |
| `properties.files` | Recommended | List of associated files |
| `properties.creators` | Recommended | Creator addresses and revenue shares |

## On-Chain Attributes Plugin

For attributes that need to be read by on-chain programs (not just displayed in wallets), use the Attributes plugin:

```typescript
import { create } from "@metaplex-foundation/mpl-core";
import { pluginAuthorityPair, ruleSet } from "@metaplex-foundation/mpl-core";

await create(umi, {
    asset: assetSigner,
    collection: collectionPublicKey,
    name: "Asset #1",
    uri: "https://arweave.net/metadata.json",
    plugins: [
        {
            type: "Attributes",
            attributeList: [
                { key: "rarity", value: "legendary" },
                { key: "power", value: "95" },
                { key: "generation", value: "1" },
            ],
        },
    ],
}).sendAndConfirm(umi);
```

On-chain attributes are useful when your program logic needs to read NFT properties (e.g., staking rewards based on rarity).

## Updating Metadata

The update authority can change metadata:

```typescript
import { update } from "@metaplex-foundation/mpl-core";

await update(umi, {
    asset: publicKey("ASSET_ADDRESS"),
    name: "Updated Name",
    uri: "https://arweave.net/new-metadata.json",
}).sendAndConfirm(umi);
```

To update the off-chain metadata, upload a new JSON file and update the URI.

## Metadata for Different Content Types

### AI-Generated Image

```json
{
    "name": "Sunset Valley #7",
    "description": "AI-generated landscape",
    "image": "https://arweave.net/image.png",
    "attributes": [
        { "trait_type": "Prompt", "value": "sunset over a valley, oil painting style" },
        { "trait_type": "Model", "value": "DALL-E 3" },
        { "trait_type": "Style", "value": "Oil Painting" }
    ]
}
```

### Music / Audio

```json
{
    "name": "Beat #23",
    "description": "AI-generated lo-fi beat",
    "image": "https://arweave.net/cover-art.png",
    "animation_url": "https://arweave.net/audio.mp3",
    "attributes": [
        { "trait_type": "Genre", "value": "Lo-Fi" },
        { "trait_type": "BPM", "value": "85" },
        { "trait_type": "Duration", "value": "3:42" }
    ],
    "properties": {
        "category": "audio",
        "files": [
            { "uri": "https://arweave.net/audio.mp3", "type": "audio/mpeg" }
        ]
    }
}
```

---

**Key Takeaways**
- Metadata JSON follows a standard that wallets and marketplaces understand
- `image` and `animation_url` point to files on Arweave (permanent storage)
- `attributes` are trait/value pairs displayed in wallets
- On-chain Attributes plugin is for data that programs need to read
- The update authority can change metadata; consider revoking it for immutability
